보안 아키텍처(ITU-T X.805)

2002년 벨 연구소(Bell Labs)의 연구원들은 단순한 의문을 가지게 되었습니다.

“차 세대 네트워크의 보안을 위해 어떤 표준을 사용해야 할까?” 이어서 두 번째 질문을 하게 되었습니다. “어느 정도의 보안이 충분하고, 보안성은 어떻게 측정해야 할까?” 이에 대한 문헌과 자료를 찾아보았지만 충분하고 적합한 답을 구할 수 없게 되자 그들은 직접 보안 프레임워크를 만들게 되었고, ITU-T 기관의 보안 표준1으로 채택되기에 이르렀습니다.

표준의 서문은 다음과 같습니다. 서문의 내용은 보안 시스템 자체가 아닌 네트워크 보안 설계에 초점이 맞춰져 있는 것을 알 수 있습니다.

The telecommunications and information technology industries are seeking cost-effective comprehensive security solutions. A secure network should be protected against malicious and inadvertent attacks and should have high availability, appropriate response time, reliability, integrity, scalability, and provide accurate billing information. Security capabilities in products are crucial to the overall network security (including applications and services). However, as more products are combined to provide total solutions, the interoperability, or the lack thereof, will define the success of the solution. Security must not only be a thread of concern for each product or service but must be developed in a manner that promotes the interweaving of security capabilities in the overall end-to-end security solution. To achieve such a solution in a multi-vendor environment, network security should be designed around a standard security architecture.

이 표준에서는 시스템의 보호대상 식별 방법으로 계층 별 분류 방법으로 3계층(인프라, 서비스, 응용)으로 분류하거나 행위 별 분류 방법으로 3가지(관리, 제어, 사용자)로 보안 프레임워크를 구분합니다.

식별된 보호대상에 대하여는 8가지의 보안 단위기술(Security Dimensions)를 적용하여 보안위협2및 취약성 분석을 하고, 정보보호 목표 또는 요구사항을 도출하게 됩니다.

8가지 보안단위기술은 다음과 같습니다.

단위기술 내용
1. 접근 제어/권한 통제(Access Control) 허가 받지 않은 사용자, 장비, 네트워크/정보시스템/서비스에 대한 접근을 제한합니다.
2. 인증(Authentication) 정보자산에 접근하고자 하는 사용자와 장비에 대해 정보 자산의 중요도에 따라 차별화된 식별성을 보장합니다.
3. 부인방지(Non-repudiation) 사용자 및 장비의 정보자산에 대한 접근/처리 내역을 보존하여 감사/추적성 및 모니터링 증적을 확보합니다.
4. 데이터 기밀성(Data Confidentiality) 정보의 허가 받지 않은 유출 로부터 보호합니다.
5. 통신보안(Communication) 정보가 인가된 종단 사이에서만 통신이 되도록 보장합니다.
6. 데이터 무결성(Data Integrity) 정보가 인가받지 않은 자에 의해 수정, 삭제, 복제되지 않도록 합니다.
7. 가용성(Availability) 인가된 사용자가 정보시스템 및 정보 자원을 필요로 할 때 접근하고 사용할 수 있도록 보장합니다.
8. 개인정보보호(Privacy) 개인정보의 유출 및 오, 남용 방지를 위해 주요 경로에 대한 보호대책을 적용합니다.

상기 보안 참조 모델에 따라 KISA(한국인터넷진흥원)에서 제시하는 보안 아키텍처 모델 은 다음과 같습니다.

보안 계층(Security Layer)는 5단계로 구분하였는데, 인프라시스템(서버, 네트워크, 시설), 어플리케이션, 데이터(DB), 단말기(Client), 인터페이스로 구분하였습니다. 상기 8개의 보안단위기술은 6가지(인증, 접근통제, 데이터보호, 암호화, 감사 추적, 위협 탐지)의 보안요소로 적용합니다. 이와 더불어 영역별 보안기능을 정리하여 서비스 형태 별로 정보자산의 특성에 따라 아키텍처 모델을 설계할 수 있도록 합니다.

이 때 모델에 의해 도출된 보안 목표 기능(Security Objectives)는 목표 시스템의 정보보안을 구현하기 위한 통제대책(Security Measures)으로 침입탐지, 침입차단, 백신, DB암호화, DB접근통제 등을 의미하는 것으로서 정보보호 솔루션으로 보안기능을 구현할 수 있습니다.

예를 들면, 데이터 보안 계층(Security Layer)에서 6가지 보안 요소 순으로 보안 기능을 도출해 보자. 인증(ID/PW, DB접근통제), 접근통제(DB접근통제), 데이터 보호(스팸메일 차단, 바이러스 침입차단), 암호화(암호화 저장), 감사(DB접속 로그관리), 위협탐지(정보유출방지)으로 도출이 가능합니다.

이와 같이 보안 계층 별, 보안 요소를 적용하여 각각 취약점 분석 및 보안 위협에 따라 정보보호 기술 요구항목을 도출할 수 있습니다.

상기 보안요소(Security Elements)는 암호 기술을 기반으로 한 인증, 접근 및 권한 통제, 부인 방지, 개인정보보호 등의 기술로서 하기에 보다 상세히 기술하고자 합니다.
